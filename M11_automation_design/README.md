# M11 · 에이전트 자동화 설계 — 절차와 계획 노하우

## 실습 목표

에이전트로 업무를 자동화하기 전에 작업을 분해하고 계획을 세우는 절차를 익혀, 토큰 낭비와 시행착오를 줄입니다.

## 핵심 개념

- 자동화 구조 설계 5단계: 목표 정의 → 작업 분해 → 역할 배분 → 계획 수립 → 실행·검증
- 역할 배분 기준
  - 판단·추론·비정형 → **AI (LLM)**
  - 반복·정밀·결정적 → **코드/스크립트**
  - 외부 데이터·서비스 → **MCP / API** (M8·M10)
  - 큰 독립 작업 분담 → **Sub-agent** (M9)

## 실습 · 마스터 프롬프트

자동화하고 싶은 업무 1개를 정하고, 아래 프롬프트로 "바로 실행"이 아니라 "분해 → 계획 → 단계 실행" 구조를 지시합니다.

```text
[목표] 무엇을 자동화할지 한 문장으로.
[제약] 사용할 도구/언어, 건드리면 안 되는 것.

바로 코드를 짜지 말고, 다음 순서로 진행해줘.
1) 이 작업을 단위 작업으로 분해하고, 각 단계를 'AI가 할 일'과
   '코드가 할 일'로 나눠서 표로 보여줘.
2) 실행 계획(순서·의존관계·각 단계 검증 방법)을 제시해줘.
3) 내가 "진행" 하면, 그때부터 1단계씩 실행하고 단계마다 결과를
   확인시켜줘. 각 단계 완료 후 다음으로 넘어가기 전에 멈춰줘.
출력은 표 위주로, 불필요한 설명은 줄여줘.
```

## 완료 확인

- [ ] 자동화하고 싶은 업무 1개를 한 문장으로 정의했다.
- [ ] 마스터 프롬프트로 계획(분해·역할배분)을 먼저 받아봤다.
- [ ] 'AI가 할 일'과 '코드가 할 일'이 나뉘었는지 확인했다.
- [ ] 승인 후 단계별 실행과 체크포인트를 확인했다.
