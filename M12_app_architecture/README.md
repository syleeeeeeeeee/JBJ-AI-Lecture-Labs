# M12 · 앱은 어떻게 작동하나 — API·DB·프론트엔드·백엔드

## 실습 목표

웹앱이 어떤 구성요소로 이뤄지고 서로 어떻게 연결되는지 큰 그림으로 이해합니다. 이를 통해 에이전트에게 "무엇을 만들어 달라"를 정확히 지시할 수 있게 됩니다.

## 핵심 개념 — 앱의 4대 구성요소 (식당 비유)

| 구성요소 | 역할 | 식당 비유 |
|---|---|---|
| 프론트엔드 | 사용자가 보고 조작하는 화면 | 손님이 앉는 홀·메뉴판 |
| 백엔드 | 요청을 처리하는 서버·로직 | 주방 (요리 담당) |
| 데이터베이스(DB) | 데이터를 저장·조회 | 식재료 창고 |
| API | 구성요소를 잇는 약속된 창구 | 주문 전표 (홀↔주방) |

전체 흐름: 프론트엔드에서 요청 → API로 백엔드에 전달 → 백엔드가 DB 조회/외부 API 호출 → 결과를 프론트엔드가 화면에 표시.

> 보안 포인트 · API 키·비밀번호 같은 민감정보는 백엔드에 보관합니다. 프론트(화면)에 두면 누구나 볼 수 있어 위험합니다.

## 실습 · 사례 분석

내가 아는 앱(쿠팡, 배민 등) 하나를 골라 4요소로 분해해 봅니다. 모든 앱이 DB를 쓰는 것은 아니며, 외부 API에서 실시간으로만 데이터를 가져오는 앱은 자체 DB가 없을 수도 있습니다.

## 완료 확인

- [ ] 내가 아는 앱 하나를 골라 4요소(프론트엔드·백엔드·DB·API)로 분해해 봤다.
- [ ] 그 앱에 DB가 필요한지, 외부 API만으로 되는지 판단했다.
- [ ] "검색 클릭 → 화면 표시" 흐름을 4요소로 설명할 수 있다.
